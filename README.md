
# Clean Eye: AI 기반 실시간 웹 콘텐츠 필터링


## 프로젝트 소개 (Introduction)

Clean Eye는 저연령층, 고연령층 등 인터넷 사용자들이 웹 서핑 중 유해 콘텐츠에 노출되는 문제를 해결하기 위해 시작된 AI 기반 실시간 콘텐츠 필터링 시스템입니다.

Chrome 브라우저 확장 프로그램으로 제작되어, 사용자가 보고 있는 웹페이지의 텍스트와 이미지를 실시간으로 분석하고 유해한 내용은 가리거나 순화된 표현으로 변경하여 보다 안전하고 쾌적한 인터넷 환경을 제공합니다.

## 주요 기능 (Key Features)

* 🤖 AI 기반 실시간 필터링: 텍스트는 Google Gemini, 이미지는 YOLOv8 모델을 사용하여 콘텐츠의 유해성을 실시간으로 판별합니다.

* 🌐 크롬 확장 프로그램: Chrome, Edge 등 주요 브라우저에서 간편하게 설치하고 사용할 수 있으며, 웹페이지의 DOM에 직접 접근하여 콘텐츠를 제어합니다.

* 🚀 하이브리드 텍스트 분석: 1차로 자체 DB에서 알려진 유해 단어를 신속하게 필터링하고, 2차로 AI가 문맥을 파악하여 신조어나 우회적인 표현까지 탐지하여 정확도를 높였습니다.

* 🔧 사용자 설정: 사용자가 필터링 강도를 직접 조절하여 자신에게 맞는 필터링 수준을 설정할 수 있습니다.

## 시스템 아키텍처 (Architecture)

Clean Eye의 필터링 과정은 다음과 같습니다.

1.  콘텐츠 추출 (Client): 사용자가 웹페이지에 접속하면, 클라이언트(확장 프로그램)가 페이지의 모든 텍스트와 이미지 URL을 추출합니다.
2.  API 요청 (Client → Server): 추출된 데이터를 REST API를 통해 텍스트 서버와 이미지 서버로 전송합니다.
3.  콘텐츠 분석 (Server):
    * 텍스트 서버는 DB와 Gemini AI 모델을 통해 텍스트의 유해성 점수를 계산합니다.
    * 이미지 서버는 전송된 URL의 이미지를 다운로드하여 YOLO AI 모델로 유해 요소를 탐지합니다.
4.  결과 반환 (Server → Client): 서버는 분석 결과를(필터링할 단어 목록, 검열 처리된 이미지 등) 클라이언트에 반환합니다.
5.  필터링 적용 (Client): 클라이언트는 반환된 결과를 바탕으로 웹페이지의 HTML 코드를 직접 수정하여 사용자에게 필터링된 화면을 보여줍니다.


## 개발 과정에서의 문제 및 해결 (Challenges & Solutions)

* Challenge 1: 브라우저 보안 정책 (HTTP 요청 제한)
    * Chrome의 보안 규정으로 인해 확장 프로그램이 웹에서 직접 HTTP 요청을 보낼 수 없는 문제가 있었습니다.
    * Solution: 요청을 `Background` 스크립트로 우회시켜 서버와 통신하는 방식으로 해결했습니다.

* Challenge 2: 서버 성능 및 비용 문제
    * 예산 제약으로 고성능 AWS 서버를 사용하기 어려워 AI 모델의 응답 속도가 느려지는 문제가 발생했습니다.
    * Solution: 서버에 DB 캐시를 구축하여, 이전에 요청된 적 있는 동일한 콘텐츠는 AI 모델을 다시 호출하지 않고 캐시된 결과를 즉시 반환하도록 하여 응답 속도를 개선하고 API 비용을 절감했습니다.
